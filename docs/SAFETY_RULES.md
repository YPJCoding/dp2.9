# 安全规则

## 默认关闭

所有业务功能默认关闭。

模板中的示例功能也默认关闭，避免新项目复制后误启用未完成逻辑。

## 风险分级

### 低风险

- 只读查询。
- 纯日志。
- 纯菜单展示。
- 不改变玩家、数据库、内存、经济数据的辅助功能。

低风险功能仍然需要配置开关和日志。

### 中风险

- Hook 游戏流程但不修改经济数据。
- 修改玩家状态但可快速恢复。
- 依赖固定地址的读取逻辑。

中风险功能必须有重复 hook 防护和测试清单。

### 高风险

以下能力属于高风险：

- 发物品。
- 发点券 / 代币 / SP / TP / QP。
- SQL update / delete。
- shell 命令。
- 清理背包。
- 修改角色状态。
- 内存 patch。
- Frida hook。
- 经济系统相关逻辑。

高风险功能必须：

1. 默认关闭。
2. 独立 risk 开关。
3. GM-only 或明确权限控制。
4. 完整日志。
5. 测试服验证后才能启用。
6. 有明确回滚方式。

## SQL 规则

禁止：

- 通用 SQL 执行器。
- 用户输入直接拼 SQL。
- 无白名单字段 update。
- delete 语句默认开启。
- 在 hook 热路径中执行 SQL。

允许：

- 固定 SQL 模板。
- 所有动态值来自 `tonumber` 后的数字或白名单枚举。
- 有独立 risk 开关。
- 有完整日志。

## Shell 规则

禁止：

- 通用 shell 执行器。
- 用户输入拼进 shell。
- 默认开启 shell 功能。
- 在 hook 热路径中执行 shell。

确需 shell 时必须：

1. 固定脚本路径。
2. 参数白名单或数字化。
3. 独立 risk 开关。
4. 完整日志。
5. 测试服验证。

## JS/Frida 规则

禁止：

- 无注释真实地址。
- 无重复 hook 防护。
- 无配置开关的 hook。
- 入口文件直接堆大量业务逻辑。
- 在业务模块中散落裸地址。
- 默认开启内存 patch。

要求：

- 地址集中管理。
- 每个地址写清楚来源、版本、用途、风险。
- 每个 hook 有防重复变量。
- Hook 内使用 try/catch 防止异常扩散。
- 高风险 patch 单独模块、单独开关、单独文档。

## GM 指令规则

GM 指令必须：

- 默认关闭。
- GM-only。
- 参数严格校验。
- 命中后不透传。
- 未命中时透传。
- 拒绝、失败、成功都记录日志。

禁止：

- 非 GM 可用。
- 任意玩家输入直接影响经济。
- 通用 SQL / shell 命令。
- 不记录日志的高风险操作。

## 文档规则

新增高风险功能时必须同步补充：

- 功能说明。
- 配置说明。
- 风险说明。
- 测试清单。
- 回滚方式。
