# 动态路线图

本文档用于跟踪 `dp2.9` 当前重构方向，避免 TODO 随着实现变化而偏航。

## 1. 当前结论

当前分支已经从“只做文档和草稿”推进到了“运行入口已模块化”的阶段。

关键事实：

- `df_game_r.lua` 已经瘦身为轻量入口。
- 所有 `item_handler` 旧实现已经从 `df_game_r.lua` 移除。
- 所有 handler 模块已经通过 `script/bootstrap.lua` 接入加载链路。
- `script/config.lua` 已经开启全部 handler 模块。
- SQL、删除、shell 类高风险能力仍默认关闭。

因此，后续 TODO 不能继续只按旧的 P0-P7 文档阶段推进，需要拆成两个目标：

1. 安全可部署版。
2. 完全功能恢复版。

## 2. 目标 A：安全可部署版

目标：服务器能启动，低/中风险功能可用，高风险功能默认拒绝并返还道具。

当前估算进度：约 70%。

还缺：

- [ ] 服务器普通频道启动验证。
- [ ] 确认 `bootstrap` 日志正常。
- [ ] 确认 `frida.load` 正常。
- [ ] 确认 `UseItem2` hook 正常。
- [ ] 验证低风险 handler：任务清理、异界重置。
- [ ] 验证中风险 handler：觉醒、转职任务、继承、跨界。
- [ ] 验证高风险 handler 默认拒绝并返还道具。
- [ ] 根据服务器日志修正 require 路径或 API 差异。

完成后可以作为“安全默认底板”部署。

## 3. 目标 B：完全功能恢复版

目标：原 `df_game_r.lua` 中所有道具功能都能按预期工作，包括 SQL、删除、shell 类功能。

当前估算进度：约 60%。

还缺：

- [ ] 明确哪些高风险功能允许恢复原行为。
- [ ] 对 SQL 类 handler 做数据库备份和回滚方案。
- [ ] 对删除类 handler 做二次确认或白名单策略。
- [ ] 对 PVP shell handler 确认脚本路径、脚本输出和 SQL 安全性。
- [ ] 为高风险功能补操作日志。
- [ ] 在测试服逐项开启风险开关验证。
- [ ] 决定正式服默认是否保持关闭。

完成后才能称为“所有功能完全恢复”。

## 4. 当前偏航点

### 4.1 “尽量不改变行为”已经不完全准确

当前重构为了安全，已经把 SQL、删除、shell 类功能默认改为关闭。

这意味着：

- 对安全可部署版，这是正确方向。
- 对完全功能恢复版，还需要后续开启和测试风险开关。

### 4.2 P4 JS 审查不应优先于服务器启动验证

`df_game_r.js` 审查仍重要，但当前最大的未知风险是 Lua 入口能否在真实服务器加载：

- Lua require 路径
- bootstrap 加载
- handler 注册
- DPX API 调用
- Frida 回调

因此下一步优先级应从“继续写文档/审查 JS”调整为“服务器启动验证”。

### 4.3 README TODO 需要继续保留，但不应作为唯一进度依据

README 的 P0-P7 适合记录重构任务；本路线图用于记录部署验收。

后续每完成一步，应同时更新：

- README TODO
- 本路线图
- 必要时更新 CHANGELOG

## 5. 建议接下来的执行顺序

### Step 1：代码自检

- [ ] 修正过时注释，例如 `bootstrap.lua` 仍称自己为模板。
- [ ] 检查 `script/config.lua` 是否需要恢复说明注释。
- [ ] 检查模块注册顺序是否需要固定。
- [ ] 检查所有 handler 是否都有日志和失败返还。

### Step 2：服务器烟测

- [ ] 部署到测试服或普通频道。
- [ ] 启动 `df_game_r`。
- [ ] 观察 bootstrap 日志。
- [ ] 登录角色验证上线提示。

### Step 3：低/中风险功能测试

- [ ] 任务清理券。
- [ ] 异界重置券。
- [ ] 觉醒券。
- [ ] 转职任务券。
- [ ] 继承券。
- [ ] 跨界石。

### Step 4：高风险默认关闭测试

- [ ] SQL 类功能应默认拒绝并返还。
- [ ] 删除类功能应默认拒绝并返还。
- [ ] PVP shell 功能应默认拒绝并返还。

### Step 5：按需恢复高风险功能

仅在测试服开启：

```lua
risk = {
    enable_sql_handlers = true,
    enable_delete_handlers = true,
    enable_shell_handlers = true,
}
```

逐项验证后再决定正式服默认值。
